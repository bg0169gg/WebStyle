GitHub

This guide will teach you how to setup your environment for Web Development, the tools you would need and guide you trough your first website.

First, what is Git and GitHub?

Git is a version control system used to track changes in code and collaborate with others on software development projects. GitHub is a cloud-based platform owned by Microsoft that hosts Git repositories, providing tools for version control, collaboration, and project management. Git is used for managing code versions and maintaining a history of changes, while GitHub facilitates sharing, reviewing, and collaborating on code with others, making it easier for developers to work together on projects.

Why should you use GitHub to host your web project?

GitHub offers free hosting for static websites through GitHub Pages, making it a convenient and reliable platform for saving, managing and hosting your website project, while providing version control, enabling easy tracking of changes and collaboration with other people or colleagues.

How to push your web project on GitHub and set up web hosting?

Step 1. Getting required software Because of its simple UI/UX, we recommend you use GitHub Desktop (Windows, macOS) for managing Git repositories locally and on the web. You can download it from the official GitHub website, or if you prefer terminal-based work, you can install the official Git client (Windows, macOS, Linux), allowing you to manage your repositories from Visual Studio code.
After installing GitHub Desktop, log in and configure your profile settings. If you choose Git, you would need to set your name and your GitHub's email by using:
git config --global user.name "Your Name" git config --global user.email "your_email@example.com"
If you want to learn how to create, clone, pull, fetch and push repositories, read the official Git documentation and instructions.

Step 2. Create a new repository and push your files into it From the top left corner of the titlebar select the File menu and click on 'New repository'. Make a name, preferably the title of your project, description and location of the local folder. If you plan for people to see, and make copies of your repository, check 'Initialize with README' and select a license that fits your requirements, by helping yourself with the open-source license page choosealicense.com.
Note: Git ignore is a simple file that defines local files and folders which shall not be tracked and uploaded (ignored). If you recognize any of your editorsor languages which you use, select it. Otherwise, leave it blank, as it may interfeer with your file configuration.
After the repository has been created, click on the undo button at the bottom of the sidebar, to undo the default initial commit. Then click on Reveal in File Explorer/Finder (file manager depends on the OS you use). If you have started working on your project, copy the files inside the folder, otherwise make a new file structure as you like. Note that Git does not track and upload empty folders, so dont be confused that you doont see your folder if its empty. This is also the perfect time to take a look at that README.md file and fill it with simple info, as name, description, used libraries, contributions and credits.
After you're done with your files, look at the bottom of GitHub Desktop's sidebar. As the summary, you can put something like 'Initial commit', or 'First push', the description as you like, and before clicking commit, take a look if all of your files/folders are in the treeview above. If everything is right, commit, wait for it to save the configuration, and you're done!

Step 3. Publish repository and set up webhost After commiting you should see the repo's home screen. Click on the blue 'Publish to GitHub' button and follow the steps, deselecting 'keep private', so others can view it and see the website. After it uploads successfully, click on 'Show in Github', click on Settings from the top navbar and go to Pages. From there, under branch, select your main branch (main, master, etc.), select the root of the page (usually just /root) and click save.
From now on, you can find your project on *your_username*.github.io/*your_repo_name*

That's it! From now on after every push you should be able to view your site on your repository's domain.
